Migrating the Cardemo Application

The following files comprise the Cardemo application. Files that required modification for ajaxification are listed in red and accompanied by an explanation of the change. New files are listed in blue and accompanied by a description of the content of that file.

File Change Description
bottomMatter.jsp  
carDetail.jsp Add <jsfExt:ajaxZone> around car pricing information. Also remove "recalculate" button since recalculation is done dynamically.
chooseLocale.jsp  
confirmChoices.jsp  
customerInfo.jsp  
devtime_js.html JavaScript functions required for the ajaxZone in the optionsPanel.jsp page. Click here to view the file.
finish.jsp  
index.jsp Change to use prefix mapping for this application due to a current limitation in Shale Remoting.
optionsPanel.jsp This file is dynamically included on carDetail.jsp. It contains the markup for the options panel that allows the user to configure their options for the car of their choice. This page was changed to include an <jsfExt:ajaxZone> around the portion of the panel that is to be subject to JSF-Extensions Dynamic Updating.
storeFront.jsp  
stylesheet.css  
WEB-INF/carstore.tld  
WEB-INF/faces-config.xml Add <resource-bundle> element, new in JSF 1.2. This element allows AJAX request to benefit from the Faces Localization scheme formerly only available to users of the <f:loadBundle> tag. Without this feature, data sent via AJAX would not always be localizable.
WEB-INF/web.xml Change to server side state saving (not required). Change to use prefix mapping for FacesServlet. This is required due to a limitation of the current Shale Remoting implementation. Add custom lifecycle init-param, as required for DynaFaces.
